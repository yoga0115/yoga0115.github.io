The Fourier Transform（傅葉爾轉換）

在 OpenCV 中所做的圖像或影像處理幾乎都會用到 Fourier Transform 的概念，及內容大概是說：所有的波形都是一群不同頻率的正弦函數的總和。
這個概念使得OpenCV在圖像的操作上可以運行，因為透過不同的訊號（可能是圖像的Pixels）可以感測出圖型的區塊等等。

Threshold

  1. 決定一個 threshold 可以使整幅圖像分成了非黑即白的二值圖像
  2. cv2.threshold(image, 進行分類的閾值, 高於（低於）閾值時賦予的新值, 方法選擇參數)
  3. 該函數有兩個返回值，第一個retVal，第二個就是處理後的圖像。

  1. Opencv提供的一個通用的2D濾波函數為cv2.filter2D()，其運算過程即為卷積運算：
    使用一個kernel，對圖像的濾波操作過程為：將kernel放在圖像的一個像素A上，求與之對應的圖像上的每個像素點的和，kernal不同，得到的結果不同
    [補] kermel是應用於圖像中的區域以在目標圖像中生成單個像素的一組權重。
  2. cv2.blur 和 cv2.boxFilter 是用在 filter 的 kernel 為均值時
  3. cv2.GaussianBlur()是把 kernel 換為一組符合高斯分佈的數值

HoughLines and HoughCircle 用來偵測直線與圓

hough變換原理介紹：
http://docs.opencv.org/3.0-beta/doc/py_tutorials/py_imgproc/py_houghlines/py_houghlines.html

hough變換在算法設計上可以如下步驟：(截自：http://blog.csdn.net/on2way/article/details/47028969)

  1. 將參數空間(ρ,θ)量化，賦初值一個二維矩陣M，M(ρ,θ)就是一個累加器了。
  2. 然後對圖像邊界上的每一個點進行變換，變換到屬於哪一組(ρ,θ)，就把該組(ρ,θ)對應的累加器數加1。
  3. 當所有點處理完成後，就來分析得到的M(ρ,θ)，設置一個閾值T，認為當M(ρ,θ)>T，就認為存在一條有意義的直線存在。而對應的M(ρ,θ)就是這組直線的參數。
  4. 有了M(ρ,θ)和點（x,y）計算出來這個直線就ok了。

HoughLine

HoughLine 相關的檢測函數有 cv2.HoughLines() and cv2.HoughLinesP()
cv2.HoughLines 的運算比較精確，cv2.HoughLinesP 比較節省運算資源
cv2.HoughLines returns an array of (ρ,θ)
cv2.HoughLines has four inputs: img(binary), ρ and θ accuracies, threshold(T)
