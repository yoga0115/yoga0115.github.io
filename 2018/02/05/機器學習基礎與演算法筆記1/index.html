機器學習概論

What is Machine Learning

Human Learning > Manual Programming > Machine Learning

Manual Programming 是經由人類學習之後創造的規則寫成程式來讓機器運作，但世界上充滿了雜訊（Noise）也就是所謂的例外狀況，
這些雜訊相對於一般狀況（訓練資料）可能很久才會出現一次，Machine Learning 的出現就是為了能有效處理這些雜訊。

When to manual program?

  • 有固定不變的結果的問題
  • Problems lacking data
  • Problems with easily separable data

When to use machine learning?

  • 有雜訊的資料
  • 高維度資料
  • 資料量很大
  • 資料可能有時間序問題

A simple & usable map for ML


Clustering : 不是很確定有什麼樣的標籤，是在資料探索完之後透過聚類去發現的
Classification : 已經有明確的標籤，所以使用監督是學習的方式

監督是學習中要思考的幾個問題

  1. 資料是如何分布以及要用什麼樣的取樣策略？
  • 如果取樣的資料太多偏頗，容易下出不好的結論
  • 要一次就取到最好的樣本幾乎是不可能的，所以通常會取很多次，當取樣次數夠多的時候越能接近真實的分佈情況
  1. 我們要如何表示特徵？
  2. 如何把資料給分群？這裡的分群指的是訓練、驗證及測試資料
  3. 訓練出的模型該如何衡量？
  4. 如何在應用場域中如預期般表現？

特徵

在做特徵決定的時候，常常因為主觀印象而有偏頗
舉個例子: 區分棒球及網球時以顏色為特徵，但如果出現了黃色的棒球？
深度學習最大的貢獻，在圖像的辨識上去學習一些好的特徵

維度的詛咒

  • 高維度的空間很難去想像
    舉個例子：如果我們要看一個球體表面的某個點往球心的方向行走一小段距離，這段距離與出發點所合出的體積，他對總體機的占比為多少？結論是隨著維度的增加，佔比會越來越大，意思就是，當維度越大，球體會越趨近於一個空心的球
    所以，在高維度空間中，大部分的質量及體積都是在表面；也就是說，在高維度的資料中，在越表面的地方要去學出特徵往往是更困難的（很玄對吧，這段我也聽不太懂）
  • 高維度帶來的問題：例如人臉辨識的1:1和1:N的問題
  • 解決方法：投到一個新的空間上去看，其實某些特徵是可以用距離簡單去計算的

一個好的決策邊界（threashold) 應該要在哪裡？

True / False, Positive / Negative

  • True positive (TP): sample is an apple, classified as an apple.
  • False positive (FP): sample is not an apple, but classified as an apple.
  • True negative (TN): sample is not an apple, classified as not an apple.
  • False negative (FN): sample is an apple, but misclassified as “not an apple”.

    Precision = TP / (TP + FP)
    Recall = TP / (TP + FN)
    F-measure = 2[precision · recall / (precision + recall)]
    Accuracy (ACC) = (TP+TN) / (TP + TN + FP + FN) 容易被大量的 TN 影響

Generalize

學習出的模型是不是真的理解？還是只是將相對應的答案記起來

  • Bias (偏差) : 訓練出的東西，放到真實的情況一定會有偏差
  • Variance (多樣性) : 為了避免 Bias 所以會需要一定的多樣性
    看個圖來理解一下：
  • Fitting
    Underfitting：模型太過簡單以至於不能 fit 真實情況，有高偏差、少多樣性
    Overfitting：模型太過擬合以至於不能 fit 真實情況，有低偏差、高多樣性
  • Bias-Variance Trade-off
    Error (mean square error, MSE) = noise^2 + bias^2 + variance
    建議：寧願去增加 variance 來降低 noise

Discriminative Model：學習資料邊界，來區分資料

Generative Model：學習資料模型，來分類資料
在使用上要避免預先假設資料符合什麼樣的模型
